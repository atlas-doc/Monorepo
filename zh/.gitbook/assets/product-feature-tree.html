A. 标准机票预订流程
通用机票预订场景，从票价查询开始，经过验价、预定、可选附加服务（行李/选座），到创建订单并支付。行李接口与选座接口位置灵活，可在预定前后调用。
A1. 票价查询
POST /search.do
依赖：无（入口接口）
  • 行程类型（单程 / 往返）
  • 乘客数（成人 / 儿童 / 婴儿）
  • 出发 / 到达城市或机场 IATA 码
  • 出发 / 返回日期 (yyyyMMdd)
  • 指定航司 / 航班号（可选过滤）
  • 多运价舱位、结算币种
A2. 验价
POST /verify.do
依赖：需先调用票价查询
  • 传入 Search 的 routingIdentifier
  • 实时价格（原始价 vs 新价）
  • 返回 sessionId（用于后续下单）
  • 返回 bookingRequirement
  • 退改签规则 / 行李额度
A3. 预定
POST /order.do
依赖：需先调用验价
  • 传入 sessionId + offerId（可选）
  • 乘客信息（姓名/生日/证件/国籍/性别）
  • 支付方式选择
A4. 行李接口（位置灵活）
POST /getLuggage.do
前置：验价 或 出票履约接口；可在预定前后调用
  • 参数：offerId（sessionId / offerId）
  • 返回各航段行李产品列表
  • 含重量/件数/价格/航司原始价
  • 在 Order 中使用 productCode 添加
A5. 创建订单
POST /order.do
依赖：需先调用验价；可携带行李/选座 productCode
  • 传入 sessionId + offerId（可选）
  • 乘客信息（姓名/生日/证件/国籍/性别）
  • 附加行李 / 选座 (productCode)
  • 支付方式选择
A5-1. 选座接口
POST /seatAvailability.do
前置：验价 或 出票履约接口
  • 模式：独立模式 / 非独立模式
  • 参数：offerId + carrier + 航段信息
  • 返回座位图及可用座位
  • 在 Order 中使用 productCode 添加
A5-2. 行李接口
POST /getLuggage.do
可在创建订单阶段再次调用以确认行李产品
  • 参数：offerId（sessionId / offerId）
  • 返回各航段行李产品列表
  • 含重量/件数/价格/航司原始价
  • 在 Order 中使用 productCode 添加
A6. 订单确认（仅 FR）
POST /orderCommit.do
仅 FR 集成流程，需先 Order
  • 获取 FR 确认页链接
  • 用户确认后进入支付
A7. 支付
POST /pay.do
依赖：需先调用创建订单
  • Deposit（预存款）支付
  • VCC（虚拟信用卡）支付
  • MoR（商户代收）模式
  • 信用卡：Visa / MC / AE / Discover
A8. 订单查询
POST /queryOrderDetails.do
依赖：需先调用创建订单
  • 按订单号查询
  • 按 PNR + 航司查询
  • 组合查询（订单号 + PNR + 航司 + 参数）
  • 返回完整订单详情
B. 出票履约接口流程
通过出票履约接口直接获取报价，经验价后，依次调用行李接口、选座接口，再创建订单并支付。
B1. 出票履约接口（获取报价）
POST /getOffers.do
前置依赖：无
  • 与 Verify 的区别：不依赖 Search
  • 提高价格验证成功率，增加 L2B
  • 需联系客户经理开通
参数：乘客数 + 去程航段(航司/机场/航班号/日期) + 返程航段(可选)
B2. 验价
POST /verify.do
前置：出票履约接口
  • 实时价格确认
  • 返回 sessionId（用于后续下单）
  • 退改签规则 / 行李额度
B3. 行李接口
POST /getLuggage.do
前置：验价 或 出票履约接口
  • 参数：offerId（sessionId / offerId）
  • 返回各航段行李产品列表
  • 含重量/件数/价格/航司原始价
  • 在 Order 中使用 productCode 添加
B4. 选座接口
POST /seatAvailability.do
前置：验价 或 出票履约接口
  • 模式：独立模式 / 非独立模式
  • 参数：offerId + carrier + 航段信息
  • 返回座位图及可用座位
  • 在 Order 中使用 productCode 添加
B5. 创建订单
POST /order.do
前置：验价；可携带行李/选座 productCode
  • 使用 offerId 代替 verify 的 sessionId
  • 乘客信息（姓名/生日/证件/国籍/性别）
  • 附加行李 / 选座 (productCode)
  • 支付方式选择
⚠️ 注意：使用 offerId 代替 verify 的 sessionId
B6. 支付
POST /pay.do
依赖：需先调用创建订单
  • Deposit（预存款）支付
  • VCC（虚拟信用卡）支付
  • MoR（商户代收）模式
  • 信用卡：Visa / MC / AE / Discover
B7. 订单查询
POST /queryOrderDetails.do
依赖：需先调用创建订单
  • 按订单号查询
  • 按 PNR + 航司查询
  • 组合查询（订单号 + PNR + 航司 + 参数）
  • 返回完整订单详情
C. 出票后附加服务流程
适用前提：订单已出票，出发日期在未来。目前仅支持 BAGGAGE（行李）类别。
C1. 附加服务搜索
POST /postBookingAncillarySearch.do
前置：订单已出票
  • 参数：原始订单号 + 类别(BAGGAGE)
  • 返回：sessionId + 可用行李产品列表
C2. 附加服务下单
POST /postBookingAncillaryOrder.do
前置：需先调用附加服务搜索
  • 参数：sessionId + 类别 + 订单号 + 乘客选择
业务约束
  • 同一类型行李每个航段只能加购一次
  • 婴儿不可购买行李
  • 联程航段必须使用相同产品码
  • 已出票/出票中不可重复购买
  • 往返可不同产品码
D. 可选的横向附加服务
可在多个流程（标准预订 / 出票履约）中调用的可选服务，可并行执行。行李接口与选座接口位置灵活，可根据业务需要在验价后或创建订单前后调用。
D1. 行李接口
POST /getLuggage.do
前置：验价 或 出票履约接口；位置灵活
  • 参数：offerId（sessionId / offerId）
  • 返回各航段行李产品列表
  • 含重量/件数/价格/航司原始价
  • 在 Order 中使用 productCode 添加
D2. 选座接口
POST /seatAvailability.do
前置：验价 或 出票履约接口
  • 模式：独立模式 / 非独立模式
  • 参数：offerId + carrier + 航段信息
  • 返回座位图及可用座位
  • 在 Order 中使用 productCode 添加
E. 通知与事件管理
E1. 注册 Webhook
POST /updateWebhookURL.do
  • 参数：cid + url（回调地址）
  • 注册后接收航变等异步通知
E2. 事件列表查询
POST /event/getPageList.do
  • 筛选：事件ID / 订单号 / PNR / 时间范围
  • 分页查询
email.schedulechange
航变邮件通知 — 通过邮件发送的航班变更通知
abnormal.cancelled
异常取消 — 非正常流程的订单取消事件
order.schedulechange
航变 API 通知 — 通过 API 发送的航班变更通知
F. 出票后操作流程
出票后的退票处理、订单维护及 PNR 认领与提取操作。
F1. 退票处理 3 API
退票询价
POST /refundQuotation.do
查询退票费用及可退金额，获取退票报价
退票提交
POST /refund.do
提交退票申请，处理订单退票
退票订单查询
POST /queryRefundOrders.do
查询退票处理进度及退票订单详情
F2. 订单维护 3 API
订单重新生成
POST /regenerateOrder.do
重新生成订单，用于异常订单恢复
停止出票
POST /stopTicket.do
停止正在进行的出票流程
订单列表查询
POST /orderList.do
按条件查询订单列表，支持分页与筛选
F3. PNR 认领与提取 2 API
PNR 认领
POST /pnr/claim.do
认领已存在的 PNR 到 Atlas 系统中
PNR 提取
POST /extractPnr.do
从 GDS 提取 PNR 详情信息

辅助运营与系统集成的工具类接口，涵盖余额查询、邮件查询、航线数据导出及令牌生成。
G1. 余额查询
POST /balance.do
查询账户当前余额
G2. 邮件查询
POST /queryMail.do
查询与订单关联的邮件记录
G3. 航线导出
POST /route/export.do
导出航线数据用于分析或运营
G4. ATRIP 令牌
POST /getAtripToken.do
生成 ATRIP 访问令牌
完整 API 调用链路
Atlas API 提供的 7 条完整调用链路
链路 1：标准预订 票价查询 → 验价 → 预定 → [行李接口（位置灵活）] → 创建订单 → [选座接口] → [行李接口] → [订单确认] → 支付 → 订单查询
链路 2：出票履约 出票履约接口 → 验价 → [行李接口] → [选座接口] → 创建订单 → [订单确认] → 支付 → 订单查询
链路 3：出票后加购 (已出票订单) → 附加服务搜索 → 附加服务下单
链路 4：异步事件 (已出票订单) ← Webhook ← (航变/取消事件) → 事件列表查询
链路 5：退票处理 (已出票订单) → 退票询价 → 退票提交 → 退票订单查询
链路 6：订单维护与 PNR (已出票订单) → 订单重新生成 / 停止出票 / 订单列表查询 / PNR 认领 / PNR 提取
链路 7：运营工具 余额查询 / 邮件查询 / 航线导出 / ATRIP 令牌
